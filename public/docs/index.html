Introduction

Base URL: http://localhost:8004/

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

POST api/register

Example request:
curl --request POST \
    "http://localhost:8004/api/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"first_name\": \"eligendi\",
    \"last_name\": \"velit\",
    \"phone_number\": \"error\",
    \"email\": \"fernando47@example.com\",
    \"password\": \"`;_Vj!`j#XwpM\"
}"

const url = new URL(
    "http://localhost:8004/api/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "first_name": "eligendi",
    "last_name": "velit",
    "phone_number": "error",
    "email": "fernando47@example.com",
    "password": "`;_Vj!`j#XwpM"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

POST api/upload_documents

Headers

Content-Type      

Example: multipart/form-data

Accept      

Example: application/json

Body Parameters

file   file   

Must be a file. Must not be greater than 20480 kilobytes. Example: /private/var/folders/wr/nmpm656d4d5b5klb8c61fkpw0000gn/T/phpjPsI7w

id   string   

The id of an existing record in the users table. Example: eos

type   string   

Example: utility_bills

Must be one of:
  • nin_slip
  • international_passport
  • utility_bills
  • drivers_license
  • permanent_residence_card
  • proof_of_address
  • profile_picture
